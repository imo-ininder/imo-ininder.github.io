8 Tips to Ensure Reliable and Resilient Service Deployment

Imo Liu
2023-11-28
DevOps, Kubernetes, SRE

Opening

In my previous work managing all the workloads running on Kubernetes, I discovered several key guidelines that are crucial for ensuring the reliability and resilience of applications. These 8 tips come from hands-on experience and numerous trials and errors. If you’re about to create your application, consider following these steps to set yourself up for success:

  1. Set up readiness and liveness probes:

    • Define readiness probes to indicate when the container is ready to serve traffic.
    • Define liveness probes to determine if the container is still running.
    • It’s considered a best practice for liveness probe values not to be the same as the readiness probe.

  2. Ensure more than one replica is running:

    • Set the replicas field in your deployment manifest to a value greater than 1 to ensure high availability.

  3. Set up resource requests and limits:

    • Specify resource requests and limits to ensure proper allocation of resources such as CPU and memory.

  4. Configure pod anti-affinity:

    • Use pod anti-affinity to prevent multiple instances of the same application from running on the same node.
    • Utilize preferredDuringSchedulingIgnoredDuringExecution, suggesting Kubernetes to avoid placing pods with the same label on the same node, but not as a strict rule.

  5. Setup Pod Disruption Budget (PDB):

    • Define a PodDisruptionBudget to ensure a minimum number of instances are available during disruptions like node drain or eviction.

  6. Implement Rolling Updates and Rollbacks with maxSurge and maxUnavailable:

    • Setting maxSurge to 1 and maxUnavailable to 0 will roll pods one by one during the update, ensuring that no pods are unavailable.

  7. Setup terminationGracePeriodSeconds :

  • Adjust the terminationGracePeriodSeconds according to your application’s needs. This grace period allows your application to perform necessary cleanup tasks or finish ongoing processes before it gets forcefully terminated by Kubernetes.
  1. Handle SIGTERM for Graceful Shutdown:
  • Ensure your application is properly configured to handle SIGTERM signals for a graceful termination in response to termination requests sent by Kubernetes.
  • Implement signal handling within your application to catch SIGTERM signals and perform necessary cleanup or shutdown procedures, such as saving state, releasing resources, or concluding ongoing tasks gracefully.
